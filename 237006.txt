git fetch origin - Се прави tracker за гранки кои се креирани преку GitHub
git checkout (име на гранка) - Се користи за да се префрли од една на друга гранка
git add (име на датотека) - Се користи за додавање на датотека во staging area
git commit - Се користи за правење на commit на сите датотеки во staging area
git push - Се користи за да се постават commit-натите датотеки во одалечениот репозиториум
git pull - Се користи за да се земат сите промени од одалечениот репозиториум во локалниот
git merge (име на гранка) - Се користи за да се merge-не една гранка во друга

Во чекор 20 се случи конфликт при спојување на втората гранка (3-list-tasks-sorted-by-name) конфликтот беше решен со рефакторирање на кодот во main класата за да се зачуваат промените и од двете гранки. Конфликтот го решив користејќи ја resolve conflict функцијата на github.

Во чекор 20 не се случи конфликт при спојување на гранките. Командите искористени беа git checkout master, git pull и git merge 2-find-all-completed-tasks.
